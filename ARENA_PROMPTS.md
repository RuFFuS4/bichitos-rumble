# Arena Packs — Prompt Library

> **Status**: Pack 1 (Jungle Tropic) shipped 2026-04-23. This doc holds
> the ready-to-paste prompts for the 4 remaining arena packs:
> Frozen Tundra · Desert Dunes · Coral Reef Beach · Kitsune Shrine.
>
> Each pack is 3 things generated separately:
>   - **1 skybox** (panorama 2048×1024, no char limit on prompt).
>   - **1 ground texture** (tileable 1024×1024, no char limit).
>   - **5–6 prop GLBs** (each under the user's 800-char prompt budget).
>
> The arena floor (29 collapsing fragments) stays procedural — props
> live in a ring at radius 14–22 around a circular arena of radius 12.
> A JSON placement file is generated by the game, not by the user.

## Pipeline tip (burned-in lesson from Jungle pack)

The IA generator consistently outputs millions of verts and
per-leaf primitives that resist every merge strategy. **Always add
this line to every prop prompt**, otherwise you risk models in the
60–300 MB range that even the aggressive pipeline can't rescue:

```
CRITICAL GEOMETRY CONSTRAINT: single merged mesh with ONE material.
Do NOT make each leaf / face / gem a separate mesh. Target < 3000
triangles total. Low-poly chunky shapes. Think "Fall Guys obstacle"
or "Minecraft plant", not photorealistic foliage.
```

The line eats ~200 chars of the 800-char budget — the remaining 600
are plenty for the prop description.

## Common preamble (goes on top of every prop prompt)

Copy-paste this block at the start of every GLB prop prompt:

```
For "Bichitos Rumble" cartoon arena brawler — chunky chibi style,
bold outlines, vibrant saturated colours, slight cel-shading, NO
realism. Reference mix: Fall Guys + Smash Bros trophy + Pixar.
CRITICAL GEOMETRY CONSTRAINT: single merged mesh with ONE material.
Do NOT make each leaf / face / gem a separate mesh. Target < 3000
triangles total. Low-poly chunky shapes.
Output: GLB, Y-up metres, 1024px PBR baked texture, metalness ≤ 0.3,
no skeleton, no embedded lighting. Origin at base centre, Y=0.
```

~430 chars. Leaves ~370 chars for the prop-specific body.

> **⚠ LESSON LEARNED 2026-04-24**: first-generation prompts asked for
> delicate shapes (branching coral, 5-6 fronds, cluster of shells,
> broken planks) which the IA consistently splits into one-mesh-per-
> feature, hitting 1M+ triangles even with the "single merged mesh"
> constraint in the prompt. Original prompts are kept at the bottom
> of this pack for reference; the REWRITTEN ones below are what you
> should feed the IA. They replace every "delicate" silhouette with
> a chunky low-poly alternative that forces the IA into single-mesh
> output.
>
> Rules of thumb for writing beach props:
>   - NO "branches", "fronds", "planks", "shells" (plural delicate).
>   - Use words the IA interprets as solid masses: "blob", "dome",
>     "lump", "clump", "chunk", "wedge", "nugget".
>   - Reference low-poly games directly: "like a Minecraft block",
>     "like a Fall Guys obstacle", "like Lego".
>   - Cap triangles VERY low: < 1500 for anything risky.

# Integration — what happens when each pack lands

1. Drop the PNGs in their target paths (`public/images/skyboxes/<pack>.png`
   and `public/images/arena-ground/<pack>.png`).
2. Drop the raw GLBs in `public/models/arenas/_raw/<pack>/` (gitignored).
3. Run `node scripts/optimize-arena-props.mjs public/models/arenas/_raw/<pack> --pack <pack>`.
4. Inspect output sizes; anything over ~5 MB gets a run of
   `scripts/compress-arena-textures.mjs public/models/arenas/<pack> --max-size 512`.
5. Optional — anything still over budget can be sent through
   `scripts/aggressive-simplify.mjs` with `--ratio 0.002 --error 0.3`.
6. Once all 4 packs are in, the next dev block is the **loader**:
   `src/arena-decorations.ts` reads `packId` from the seed or URL,
   loads the GLBs from `public/models/arenas/<pack>/`, swaps the
   skybox and the ground texture on the arena fragments, and reads
   placement JSON for each prop's `{ radius, angleDeg, rotY, scale }`.
